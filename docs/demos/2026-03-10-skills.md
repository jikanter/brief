# brief emit skill — Generating Claude Code Skills from Briefings

*2026-03-11T04:19:04Z by Showboat 0.6.1*
<!-- showboat-id: b85be6f1-ca34-472b-9a93-7282888a290b -->

The `brief emit skill` command transforms a `.brief.md` file into a Claude Code SKILL.md — a reusable slash command that any developer on the team can invoke. This demo walks through the full workflow: authoring a brief, emitting a skill, and installing it.

## 1. A skill-oriented brief

A `.brief.md` becomes a skill when you add `skill_name` and `skill_description` to the frontmatter. Everything else — constraints, sacred regions, assumptions — maps directly to skill instructions.

```bash
cat tests/fixtures/skill.brief.md
```

```output
---
stack: [Python 3.12, PostgreSQL 16]
context: [./docs/api-spec.yaml]
skill_name: review
skill_description: Review code changes following team standards
---

# Review code following team standards

## Constraints

### Hard
- All SQL must target PostgreSQL 16
- API backward compatibility must be maintained

### Soft
- Prefer async patterns

### Ask First
- Database schema changes

## Sacred
- `src/auth/**` — Authentication logic, SOC2 audited
- `migrations/**` — Historical migrations, never alter

## Assumptions
- [ ] Current tests cover critical paths
- [x] CI pipeline runs on every PR

## Deliverable
Clear review comments with specific file/line references and suggested fixes.
```

## 2. Emitting the skill

`brief emit skill` transforms the structured brief into a SKILL.md that Claude Code understands — imperative voice, clear rules, protected regions, and verification steps.

```bash
brief emit skill tests/fixtures/skill.brief.md
```

```output
---
name: review
description: Review code changes following team standards
---

Review code changes following team standards. This project uses Python 3.12, PostgreSQL 16.

Before starting, read these files for context:
- `./docs/api-spec.yaml`

## Rules

You MUST follow these rules:
- All SQL must target PostgreSQL 16
- API backward compatibility must be maintained

Prefer these approaches when possible:
- Prefer async patterns

Ask the user before proceeding with:
- Database schema changes

## Protected regions

Do NOT modify or suggest changes to these files:
- `src/auth/**` — Authentication logic, SOC2 audited
- `migrations/**` — Historical migrations, never alter

## Verify before proceeding

Confirm these assumptions still hold before acting on them:
- Current tests cover critical paths

## Expected output

Clear review comments with specific file/line references and suggested fixes.
```

Notice how each brief section maps to a skill concept:

| Brief section | Skill output |
|---|---|
| `skill_name` frontmatter | `name:` in YAML frontmatter |
| `skill_description` frontmatter | `description:` in YAML frontmatter |
| `stack` frontmatter | Woven into opening sentence |
| `context` frontmatter | "Read these files" pre-reading list |
| Hard constraints | "You MUST follow these rules" |
| Soft constraints | "Prefer these approaches" |
| Ask First constraints | "Ask the user before proceeding" |
| Sacred regions | "Do NOT modify" protected regions |
| Unvalidated assumptions | "Verify before proceeding" checks |
| Deliverable | "Expected output" section |

Validated assumptions are omitted — they are already confirmed and don't need re-checking on every invocation.

## 3. Installing the skill

The `--install` flag writes the skill directly to `.claude/skills/<name>/SKILL.md`, ready for Claude Code to discover.

```bash
brief emit skill tests/fixtures/skill.brief.md --install
```

```output
Installed .claude/skills/review/SKILL.md
```

```bash
cat .claude/skills/review/SKILL.md
```

## 4. Automatic name derivation

When `skill_name` is omitted from the frontmatter, the skill name is derived by slugifying the H1 goal.

```bash
brief emit skill tests/fixtures/full.brief.md | head -4
```

```output
---
name: build-real-time-collaborative-document-editor
description: Build real-time collaborative document editor
---
```

The goal "Build real-time collaborative document editor" becomes the slug `build-real-time-collaborative-document-editor`. For reusable skills, setting an explicit `skill_name: collab-editor` is recommended.

## 5. Full emit from the sample brief

The sample brief in `examples/` has no skill metadata — it emits a skill purely from its structured content.

```bash
brief emit skill examples/sample.brief.md
```

```output
---
name: redesign-event-pipeline-for-10m-events-day
description: Redesign event pipeline for 10M events/day
---

Redesign event pipeline for 10M events/day. This project uses Python 3.12, PostgreSQL 16, Kafka 3.7, GCP/k8s.

Before starting, read these files for context:
- `./docs/current-architecture.md`
- `./benchmarks/performance-baseline.csv`

## Rules

You MUST follow these rules:
- v2 API backward compatibility must be maintained
- All SQL must target PostgreSQL 16, not MySQL
- Infrastructure budget ceiling is $500/month

Prefer these approaches when possible:
- Prefer async write patterns where possible
- Favor composition over inheritance in new code

Ask the user before proceeding with:
- Database schema changes
- Adding new dependencies to requirements.txt
- Changes to the CI/CD pipeline

## Protected regions

Do NOT modify or suggest changes to these files:
- `src/auth/**` — Proprietary tenant resolution logic, legally reviewed
- `src/compliance/**` — GDPR audit trail, approved by legal
- `migrations/` — Historical migration files must never be modified

## Verify before proceeding

Confirm these assumptions still hold before acting on them:
- Bottleneck is synchronous DB writes (validate against perf baseline)
- Kafka cluster can handle 10M events/day without partition rebalancing

## Expected output

Architecture doc with decision records, implementation plan with milestones,
and working code with tests achieving >80% coverage on new modules.
```

## Summary

`brief emit skill` closes the loop between human intent and agent tooling. Write a `.brief.md` once — emit it as a CLAUDE.md section, a system prompt, an AGENTS.md block, or now a reusable Claude Code skill. Same source of truth, multiple surfaces.
